# 领域词汇

## 巡检工具

一个独立的 Web 项目，提供脚本管理和报告生成两个彼此独立的功能模块。它不主动连接客户环境，不维护客户资产或巡检历史。

## 脚本管理

用于展示、复制和下载内置巡检 Shell 脚本的模块。脚本管理不创建报告任务，也不向报告生成模块传递状态。

## 报告生成

用户选择中间件类型并粘贴 JSON 日志或上传日志文件后，执行校验、规则判定、Markdown 生成与编辑、DOCX 导出的模块。

## 巡检组件

某一种中间件专属的一组协作制品，包括 Shell 巡检脚本、JSON 数据协议、解析与校验规则、告警判定规则、Markdown 模板和 DOCX 模板。各巡检组件可以拥有不同的数据字段和报告结构。

## 标准巡检日志

由巡检组件内置 Shell 脚本产生、以前后边界标记包裹的单个 JSON 文档。JSON 以结构化指标为主，只携带支持复核所需的最小证据。

## 巡检实例

一台主机上运行的一个中间件进程及其部署配置。Tomcat 在单次采集中使用主机 IP 与进程号区分实例；该标识不用于跨月追踪。

## 集群报告边界

对分布式中间件，用户在一次报告生成操作中提交的日志集合。系统不自动推断不同日志是否属于同一个集群。

## 报告生成会话

从一次日志提交开始，到 Markdown 编辑及 DOCX 下载结束的临时处理过程。会话不形成巡检历史或资产台账。

## 无状态

不保存客户资产、巡检历史、原始日志、解析结果或已生成报告。该词不排除程序在一次报告生成会话中使用内存或临时文件。

## ReportView

位于规则分析结果与具体输出格式之间的报告内容模型。Markdown 和 DOCX 均从同一份报告内容语义生成；用户编辑后的 Markdown 是最终 DOCX 的内容来源。
